Rentflix

An MVC app that I built using Microsoft ASP .NET and Entity Framework Core, and hosted using Microsoft Azure. As a mock movie rental website, it's designed to show off my skills as a developer using .NET and RESTful API. Anyone can view a list of movies using the browse page, and by logging in and creating an account, you can (pretend to) rent movies, return them, view a list of your past and current rentals, and view and write reviews for each movie.

To log in, make/view pretend rentals and leave reviews, you can use the fictitious email "newemail@gmail.com" and the password is "Password1#", or feel free to create a new account using the registration link on the log-in page. Creating, editing, and deleting movies through the site is also possible, but those web pages are only available to users with admin permissions

Bob's Computer Repair Shop

This is a group project that I helped create for a college class, alongside two of my classmates, Hannah Del Real and Jocelyn Dupis. It's a site for a fictitious computer repair shop, a user can log in either as an admin or a regular employee and submit a work order to a MongoDB database, while those with admin permissions can access a user configuration page that allows for creating, editing, and deleting users.
My personal contributions to the project included:

  • Instantiating the database and managing the team's database design strategy
  • Writing API for the following: finding a user by ID, logging a user in, finding a user's selected security questions, and finding a work order based on a requested service
  • Implementing the UI for the User Configuration page
  • Designing the registration process, its associated form, and setting up the UI for it
  • Implementing the forgot password process
  • Creating the User Profile page

To log in and access the site's admin features, use the following on the sign-in page - Email: bob@bcrs.com Password: Adminbob1

Tasks can be moved back and forth between the "Todo" and "Done" columns by dragging and dropping them thanks to RXJS implementation
